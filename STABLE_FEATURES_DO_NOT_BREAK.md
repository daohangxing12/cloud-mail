# 稳定功能清单：禁止随意删除或重构

这份文件用于固定当前邮箱系统稳定版。后续其它 AI 或工程师接手时，必须先读完再改代码。

## 当前稳定入口

- 网页入口：`mail.ntmcn.com`
- API / 同步入口：`https://212202.xyz`
- 旧入口 `email.ntmcn.com` 不再作为主入口使用。
- 旧测试域名 `api.orz.gay`、`ppp.nte.li` 不允许重新写回前端或本地软件默认配置。

## 已稳定的前端功能

- 收件箱是 Outlook 风格三栏布局：左侧菜单，中间邮件列表，右侧邮件详情。
- 收件箱顶部必须保留搜索框。
- 邮件列表必须保留头像、发件人、标题、摘要、时间等信息。
- 邮件详情必须在右侧铺开显示，不允许退回弹窗式或单列表式。
- 收件箱下面保留域名收件箱入口。
- 菜单顺序保持：收件箱、域名收件箱、已发送、垃圾邮箱等现有结构。
- 垃圾邮箱是全局垃圾邮箱，所有域名共用。
- 全局收件箱显示非垃圾邮件，支持按域名单独查看。
- 精准拉黑必须按完整邮箱地址处理，不能自动扩大为域名拉黑。

## 已稳定的后端功能

- `/api/local-agent/*` 是本地矩阵工具依赖接口，不能删除、改名或改认证方式。
- `/api/local-agent/sub-account/token` 必须支持按邮箱确保创建子邮箱和 Token。
- `/api/local-agent/mail/list` 必须支持全局读取最近邮件，用于本地软件接码和纠错。
- `/api/local-agent/code/latest`、`/getNewTkMailCode` 相关接码链路不能删除。
- 子邮箱只允许管理域名：`feilong168.com`、`baofa.de`、`ntmcn.com`。
- 资产中心邮箱同步必须保留邮箱、TikTok 用户名、粉丝、播放量、登录状态。
- 中视频成功/被拒邮件识别和统计不能删除。

## 已稳定的本地软件依赖

本地矩阵工具会依赖邮箱系统做这些事：

- 登录前按邮箱创建子邮箱和 Token。
- 读取 TikTok 登录验证码。
- 用户名登录时，根据资产中心或全局收件箱纠错真实邮箱。
- 登录成功后同步用户名、粉丝、播放量、登录状态到资产中心。
- 发布视频或读取详情时同步账号详情。

这些接口变动会直接导致本地软件登录、接码、资产同步失败。

## 修改红线

没有用户明确要求时，禁止：

- 禁止删除全局收件箱。
- 禁止删除全局垃圾邮箱。
- 禁止删除三栏阅读布局。
- 禁止删除邮件列表搜索框。
- 禁止删除拉黑入口。
- 禁止把完整邮箱拉黑改成域名拉黑。
- 禁止删除子邮箱 Token 功能。
- 禁止删除资产邮箱批量创建子邮箱功能。
- 禁止删除扫描未建档邮箱功能。
- 禁止删除中视频邮件识别统计。
- 禁止改回旧域名、旧 API、旧云端任务代理。

## 改动要求

每次改动前必须先判断：

1. 是否影响收件箱三栏布局。
2. 是否影响全局收件箱/垃圾邮箱。
3. 是否影响精准拉黑。
4. 是否影响子邮箱 Token 接码。
5. 是否影响本地矩阵工具 `/api/local-agent/*`。
6. 是否影响资产中心同步。

只要影响其中任何一点，必须先说明风险，再执行。
